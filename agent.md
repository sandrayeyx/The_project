# Agent 工作约定

## 1. 目标
本文件用于规范新对话下agent的默认工作方式，确保上下文连续、关键决策可追溯、实验记录可复用。

## 2. 新对话启动流程
每次开启新对话后，默认按以下顺序读取：
1. `agent.md`
2. `log/` 目录下与当前任务最相关的日志，若无法判断则优先读取最近更新的日志
3. 如任务涉及参数、模型或实验配置，再补充读取 `env_config.md`、`模型.md`、对应实验目录下的配置文档

## 3. 日志记录规范
1. 每个相对独立的问题维护一个对应的 markdown 日志文件
2. 同一问题的后续推进应优先追加到原日志，而不是重复新建
3. 日志文件名保持稳定，不因修改而改名
4. 每次重要分析、方案确认、执行结果产出后，都要更新对应日志
5. 每篇日志至少包含以下内容：
   - 问题描述
   - 背景/输入信息
   - 采用方案
   - 关键决策及原因
   - 执行结果
   - 待办事项
   - 更新时间
6. 修改代码文件时，需维护一个修改文件的markdown日志文件，每次修改都要标注修改的文件名称以及修改时间

## 4. 工作约束
1. 涉及重要实现或批量修改时，先给出简短计划再执行
2. 若发现历史日志与当前代码/配置不一致，需先指出冲突再继续
3. 输出结论时，尽量同步更新到对应日志，避免结论只存在于对话中
4. 对配置文件进行增添时，需添加相应的注释，配置注释延续现有中英混合、解释默认值和何时生效


## 5. 术语说明
- “问题”：指一个相对独立、可追踪的任务或分析主题
- “关键决策”：指影响后续实现、实验路径或结论解释的重要选择
